Breadcrumbs

Breadcrumbs identify the current resource in an application

  • Begin with a ul.ui-breadcrumbs; each crumb should be in its own li as a direct descendant.
  • Breadcrumbs are typically navigation links (for example, to the parent folder in a file path), and therefore should use <a> tags (except for the final breadcrumb).
  • Each navigation breadcrumb should use .ui-breadcrumb.
  • Make a breadcrumb non-interactive with the .ui-disabled class. You should only use this state when you want to indicate that the user cannot navigate to the breadcrumb (for example, if the user does not have permission to access it). Otherwise, clicking a breadcrumb should take the user to that resource.
  • Mark the final breadcrumb .ui-breadcrumb-current for an emphasized appearance.
  • The .ui-breadcrumbs-collapsed button-like element can be used as the target for a dropdown menu containing breadcrumbs that are collapsed due to layout constraints.
  • When adding another element (such as a tooltip or popover) to a breadcrumb, wrap it around the contents of the li.
  • Folder one
  • Folder two
  • Folder three
  • File
<ul class="ui-breadcrumbs">
  <li><a class="ui-breadcrumbs-collapsed" href="#"></a></li>
  <li><a class="ui-breadcrumb ui-disabled">Folder one</a></li>
  <li><a class="ui-breadcrumb" href="#">Folder two</a></li>
  <li><a class="ui-breadcrumb" href="#">Folder three</a></li>
  <li><span class="ui-breadcrumb ui-breadcrumb-current">File</span></li>
</ul>

Buttons

Buttons trigger actions when clicked.

CSS API

Use the ui-button class to access button styles. You should implement buttons using the <button> or <a> tags rather than <div> for the purposes of HTML accessibility and semantics.

  • Make sure to include type="button" on <button> tags (use type="submit" when used in a <form>) and role="button" on <a> tags for accessibility.
  • Add the attribute tabindex="0" to make <a> tags focusable. <button> elements are focusable by default.
  • For buttons implemented with <a> tags, add tabindex="-1" to disabled buttons to prevent the user from focusing them by pressing tab on the keyboard.
  • Note that <a> tags do not respond to the :disabled attribute; use .ui-disabled instead.

Responsive button groups

Add the class ui-fill to a button group to make all buttons expand equally to fill the available space. Then add the class ui-fixed to individual buttons to revert them to their original default sizes.

Alternatively, add the class ui-fill to an individual button (instead of to the container) to expand it to fill the available space while other buttons retain their original sizes.

You can adjust the specific size of a button with the flex-basis CSS property.

Dialogs

Dialogs present content overlaid over other parts of the UI.

Terminology note
The term "modal" is sometimes used to mean "dialog," but this is a misnomer. Modal is an adjective that describes parts of a UI. An element is considered modal if it blocks interaction with the rest of the application. We use the term "dialog" to avoid confusion with the adjective.

CSS API

You can create dialogs manually using the HTML markup and ui-dialog-* classes below. However, you should use the dialog JavaScript APIs whenever possible, as they automatically generate some of this markup.

Text input groups

An input group allows you to add icons and buttons within a text input to expand its functionality. For example, you might use an input group to build a visibility toggle for a password field.

CSS API

You can place a single .ui-icon or .ui-button.ui-icon-* on either end of the input. The order is dictated by the HTML markup: an element specified before the input appears on the left edge, and vice versa. You do not need to apply sizing classes to the childrenthey inherit the size of the parent input.

Icons only

You cannot use buttons with text in the CSS API for input groups. The padding for text inputs in CSS cannot accomodate buttons whose width varies due to text content. You should use icons on buttons instead.

Conversely, the InputGroup React component does support arbitrary content in its right element.

Control groups

A .ui-control-group renders several distinct controls as one unit, squaring the borders between them. It supports any number of .ui-button, .ui-input, .ui-input-group, and .ui-select elements as direct children.

Note that .ui-control-group does not cascade any modifiers to its children. For example, each child must be marked individually as .ui-large for uniform large appearance.

Control group vs. input group

Both components group multiple elements into a single unit, but their usage patterns are different.

Think of .ui-control-group as a parent with multiple children, each of them a "control."

Conversely, a .ui-input-group is a single control, and should function like so. A button inside of an input group should only affect that input; if its reach is further, then it should be promoted to live in a control group.

Menus

Menus display lists of interactive items.

CSS API

Menus can be constructed manually using the HTML markup and ui-menu-* classes below. However, you should use the menu React components instead wherever possible, as they abstract away the tedious parts of implementing a menu.

  • Begin with a ul.ui-menu. Each li child denotes a single entry in the menu.

  • Put a .ui-menu-item element inside an li to create a clickable entry. Use either <button> or <a> tags for menu items to denote interactivity.

  • Add icons to menu items the same way you would to buttons: simply add the appropriate ui-icon-<name> class*.

  • Make menu items non-interactive with the class ui-disabled.

  • Wrap menu item text in a <span> element for proper alignment. (Note that React automatically does this.)

  • Add a right-aligned label to a menu item by adding a span.ui-menu-item-label inside the .ui-menu-item, after the content. Add an icon to the label by adding icon classes to the label element (ui-icon-standard size is recommended).

  • Add a divider between items with li.ui-menu-divider.

  • If you want the popover to close when the user clicks a menu item, add the class ui-popover-dismiss to any relevant menu items.

* You do not need to add a ui-icon-<sizing> class to menu itemsicon sizing is defined as part of .ui-menu-item.
